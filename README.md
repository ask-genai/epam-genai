# EPAM GenAI Course - Practical Task Submission

## Project Overview

This repository contains the submission for the EPAM Generative AI for Software Development course's first practical task. The project focuses on researching and documenting Generative AI applications with emphasis on AI-powered chatbots for customer support.

## Repository Contents

### 1. research.md
A comprehensive research report on Generative AI applications in software development, featuring:

- **AI-Powered Customer Support Chatbots** - Key business value, technical challenges, weaknesses, and existing implementations
- **Automated Code Generation** - How GenAI accelerates software development
- **Automated Test Generation** - Improving software quality through AI-driven QA
- **Bug Detection and Security Analysis** - Proactive security and quality measures

Each use case includes:
- Business value analysis
- Technical challenges
- Limitations and weaknesses
- Real-world implementations with links

### 2. vocabulary.md
A comprehensive glossary containing 25 essential Generative AI terms, organized into categories:

- **Core Concepts**: LLM, Prompt Engineering, Hallucination, Fine-tuning, Token, Embeddings, Context Window
- **Technical Terms**: Neural Network, Transformer Architecture, Attention Mechanism, Pre-training, RLHF
- **Business & Implementation**: API, Inference, Model Bias, Latency, Scalability
- **Application-Specific**: Chatbot, NLP, Intent Recognition, Entity Recognition, Sentiment Analysis, Knowledge Base, Model Drift, Compliance

## Key Focus: AI-Powered Chatbots

The research particularly emphasizes chatbots as they represent a transformative application of GenAI in customer support, delivering:

- **24/7 Availability** - Round-the-clock customer support without human intervention
- **Cost Efficiency** - Potential 30% reduction in support costs
- **Scalability** - Handle multiple concurrent conversations
- **Challenges** - Hallucinations, context understanding, emotional interactions
